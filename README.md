# Meteostation
Сбор данных с Repka Pi

Система мониторинга микроклимата (температура и влажность) в реальном времени. Проект включает в себя сбор данных с IoT-устройства (Repka Pi), их хранение в реляционной БД и визуализацию на фронтенде.

## Стек технологий
Backend: Go (Golang), Goroutines (для конкурентного сбора данных).

Database: PostgreSQL.

Frontend: JavaScript, Webpack, HTML5 & CSS3, Chart.js.

Infrastructure: Docker, Docker-compose, Linux (Ubuntu), Makefile.

## Архитектура системы
Проект развернут в виде двух изолированных микросервисов в Docker-контейнерах:

Backend Server (Go):
- Реализована логика высокочастотного сбора данных (1 запрос в секунду).
- Использование Goroutines для обеспечения неблокирующего получения JSON-пакетов от IoT-модуля.
- Спроектирована схема БД для хранения временных рядов (Time-series data).
- Автоматизация сборки и запуска через Makefile.

Frontend Server (JS/WebPack):
- Chart.js для визуализации данных.
- Оптимизация сборки ресурсов через Webpack.

## Запуск и управление проектом
Управление проектом автоматизировано с помощью **Makefile**. Перед первым запуском убедитесь, что на вашей системе установлены Docker и Docker Compose.

### Основные команды Makefile

#### Работа с Docker-контейнерами
- `make build` — сборка Docker-образов (с флагом `--no-cache` для чистой сборки).
- `make run` — запуск всех контейнеров в фоновом режиме (аналог `docker-compose up`).
- `make stop` — остановка контейнеров без удаления.
- `make down` — полная остановка и удаление контейнеров вместе с томами (volumes).

#### Разработка фронтенда (локально, вне контейнера)
- `make frontend-install` — установка зависимостей Node.js (npm install).
- `make frontend-build` — продакшн-сборка статических файлов через Webpack.
- `make watch` — запуск Webpack в режиме разработки с отслеживанием изменений (hot-reload).

#### Справочная информация
- `make help` — выводит список всех доступных команд с описанием.
